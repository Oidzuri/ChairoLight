# ChairoLight

ChairoLight — приложение для адаптивной фоновой подсветки монитора на Windows. Оно захватывает края изображения, преобразует их в цвета адресной LED-ленты и передаёт кадры совместимому устройству по протоколу Adalight.

Проект сделан **Oidzuri** на основе открытого проекта [Prismatik](https://github.com/psieg/Lightpack). Исходный код и производные изменения распространяются по лицензии [GNU GPLv3](LICENSE); сведения о динамических зависимостях приведены в [THIRD_PARTY_LICENSES.md](THIRD_PARTY_LICENSES.md).

> Текущий статус: **0.9.0 preview**. Перед повседневным использованием сохраните копию своего профиля.

## Что изменено в ChairoLight

- современный тёмный интерфейс и запуск сразу в системный трей;
- захват через Windows Graphics Capture и Desktop Duplication;
- исправленная обработка BGRA-кадров без синего оттенка у чёрного;
- адаптивный анализ краёв вместо одного среднего цвета;
- мягкое ограничение пересвета белого, жёлтого и красного без общей потери яркости;
- пресеты сцен «Аниме», «Игры» и «Кино»;
- автоматическая классификация сцены и умная цветокоррекция;
- расширенные настройки цветового баланса и экспериментальная калибровка камерой;
- восстановление соединения с Arduino и заставка при подключении Adalight;
- импорт и экспорт профилей.

## Требования

- Windows 10 или Windows 11, 64-bit;
- монитор в SDR для проверенного базового сценария;
- Arduino/совместимый контроллер с прошивкой Adalight;
- адресная лента с отдельным питанием, рассчитанным на её ток;
- USB-порт и корректно выбранный COM-порт.

HDR пока считается экспериментальным режимом — подробности приведены в [известных ограничениях](KNOWN_ISSUES.md).

## Быстрый запуск

1. Скачайте portable ZIP из раздела Releases и распакуйте его в обычную папку пользователя.
2. Запустите `Prismatik.exe`.
3. На странице устройства выберите Adalight, COM-порт, скорость, число светодиодов и порядок цветов вашей ленты.
4. Настройте расположение светодиодов через мастер устройства.
5. Выберите способ захвата. Для Windows 10/11 сначала попробуйте Windows Graphics Capture; при проблемах переключитесь на Desktop Duplication.
6. Сохраните профиль и включите подсветку.

Portable-архив содержит только пустой служебный `main.conf`, который включает portable-режим. В нём нет готового профиля, заранее выбранного COM-порта или других настроек компьютера автора.

## Режимы захвата

| Режим | Когда использовать | Ограничения |
|---|---|---|
| Windows Graphics Capture | основной вариант на Windows 10/11 | защищённое видео и некоторые оверлеи могут не захватываться |
| Desktop Duplication | запасной быстрый вариант для игр и рабочего стола | поведение HDR зависит от драйвера и формата рабочего стола |
| WinAPI | режим совместимости | медленнее современных способов |

Если чёрный выглядит синим, сначала убедитесь, что используется сборка ChairoLight 0.9.0 или новее, затем сбросьте старую цветовую калибровку профиля.

## Сборка на Windows

Проект собирается Qt 5, MinGW-w64 и qmake. В MSYS2 установите Qt Widgets, Qt Network, Qt SerialPort и MinGW toolchain, затем из папки `Software` выполните:

```powershell
qmake-qt5 CONFIG+=release Lightpack.pro
mingw32-make -j4
```

Тесты собираются вместе с Windows-проектом. Локальный portable-архив создаёт скрипт:

```powershell
powershell -ExecutionPolicy Bypass -File scripts/release/build-portable.ps1
```

Автоматическая Windows-сборка и тесты описаны в [GitHub Actions](.github/workflows/windows-release.yml).

## Проверка релиза

Перед публикацией необходимо пройти [матрицу ручного тестирования](docs/TEST_MATRIX.md). Автоматические тесты не могут проверить реальный HDR-монитор, конкретный драйвер видеокарты, сон Windows или физическое переподключение Arduino.

## Безопасность питания

Не питайте длинную LED-ленту от пина Arduino или USB-порта. Используйте отдельный блок питания с общей землёй контроллера и ленты. Добавьте предохранитель, если это предусмотрено вашей схемой.
